Intelligent Ground Vehicle Competition

2016 - Present
Florida Institute of Technology

Developed an autonomous robot, designed to navigate a course filled with randomly placed obstacles. The robot must navigate to several GPS waypoints, then return to its starting position within a specified time limit. The official website for the competition can be found here. Our official team website can currently be found here.

The robot's software was designed to be modular and independent. For example, if one of the components failed at some point, the entire system would not crash. Instead, it would pause execution of the other components and wait for the failed component to restart before continuing. Also, the addition of new modules and components is extremely easy, as our communications framework, based on an Advanced Message Queueing Protocol (AMQP) called RabbitMQ, is highly flexible. Rabbit MQ is designed to be a pub/sub server-client service. This allows us to write one client, and thread multiple instances each time we want another component. But even more interesting is that RabbitMQ is language independent, which allows us to write a software package using multiple languages.

The main components of the robot consist of vision/line detection, written in C++, and pathfinding, written in Java. Another component, being our motor controller code, was written in python.

Projects

Here is a list of other projects that I am working on or have completed.

Client/Server - A project completed for my Network Programming class my junior year of college. The client sends information to the server to be stored in an SQL database. The data is encoded using ASN.1 protocols before being sent across the network, and decoded when it reaches its destination. The data for this project consisted of tasks, when they were started and completed, and who has responsibility for it.
AgentSystem - A project I worked on for a job interview. A user specifies an agent they would like to start on a host. PongAgents simply wait for messages sent by PingAgents, and reply when a message is received. A PingAgent must search the local area network for all PongAgents that have started, and send a ping message to each one. Currently the network discovery feature does not work, however messages can be sent between agents and understood.
